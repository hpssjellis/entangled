11 ## Using CClient library in a Bazel project
12 
13 CClient library is a component in the entangled repository, in order to use it in a Bazel project
14 * create a new Bazel project
15 * import CClient library into the project
16 
17 First is to create **WORKSPACE** and **BUILD** files for the Bazel project, for more details, please visit [Bazel Concepts and Terminology](https://docs.bazel.build/versions/master/build-ref.html).
18 
19 The **WORKSPACE** file
20 
21 ```
22 git_repository(
23  name = "entangled",
24  commit = "ENTANGLED_COMMIT_HASH",
25  remote = "https://github.com/iotaledger/entangled.git",
26 )
27 
28 # external library build rules
29 git_repository(
30  name = "rules_iota",
31  commit = "RULES_IOTA_COMMIT_HASH",
32  remote = "https://github.com/iotaledger/rules_iota.git",
33 )
34 
35 load("@rules_iota//:defs.bzl", "iota_deps")
36 
37 iota_deps()
38 ```
39 
40 The **BUILD** file
41 
42 ```shell
43 package(default_visibility = ["//visibility:public"])
44 
45 cc_binary(
46  name = "cclient_app",
47  copts = ["-DLOGGER_ENABLE"],
48  srcs = ["cclient_app.c", "cclient_app.h",],
49  deps = ["@entangled//cclient:api",],
50  visibility = ["//visibility:public"],
51 )
52 
53 ```
54 
55 You can find an example project in: [iota_cclient_example](https://github.com/oopsmonk/iota_cclient_example)

71 ## API List
72 C Client API consists of two API sets:
73 * **Core APIs** are basic APIs in [API reference](https://docs.iota.org/docs/iri/0.1/references/api-reference).
74 * **Extended APIs** are commonly used API functions for applications.
75
